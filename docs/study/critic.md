## Answers

### 1. Final-cut FFmpeg spec (ai-moive-studio)
`concatenate_videos(mode=...)` in `ffmpeg_utils.py` has three modes; **movie composition actually runs `mode="fast"`** (`movie_video_service.py` line ~196, with `transition_type="fade"` and `transition_duration=0.5` literally commented out). Concrete specs:
- **fast**: concat demuxer file + `-c copy` (no re-encode, duplicated boundary keyframes survive).
- **crossfade**: chained `xfade=transition={type}:duration=0.5:offset={cumulative-0.5}` per pair; audio handled separately via `atrim` (first clip trimmed at tail, last at head, middles both) then `concat=n=N:v=0:a=1`; encode `libx264 -preset medium -crf 18 -pix_fmt yuv420p -c:a aac -b:a 192k -ar 44100 -movflags +faststart`, 600s timeout. Offsets computed from per-clip durations probed via ffprobe.
- **trim**: every clip after the first gets `trim=start_frame=35` (audio `atrim=start=35/fps`), then stream concat, same encode settings. Default `trim_frames=35` (~1.17s at 30fps). Any failure falls back to fast mode.
- Single video: plain file copy. Per-sentence narration clips encode at `-preset slow -crf 20 -profile:v high -level 4.2`.

### 2. VectorEngine API contract
`vector_engine_provider.py`: `POST {base}/video/create` with `{prompt, images[], model, aspect_ratio, enable_upsample: true, enhance_prompt: true}`; default base `https://api.vectorengine.ai/v1`; httpx timeout 60s (connect 20s). Create returns **`id`** (code falls back to `task_id`: `result.get("id") or result.get("task_id")`, transition_service.py line 460). Status poll `GET /videos/{id}` returns `{"id", "status", "detail": {...}}` â€” statuses observed in code: `video_generating | processing | pending` (still running; detail may carry inner status), `completed` (then `video_url` at top level, falling back to `detail.video_url`), `failed` (error in `status_data.get("error")`). Download is a direct GET of `video_url` (no auth), 300s timeout, then MinIO upload. Unknown statuses are logged and left for the next 30s sweep.

### 3. Transition ordering into the timeline
`batch_create_transitions` builds one **global ordered shot list per chapter**: scenes sorted by `order_index`, shots sorted within each scene; adjacent same-scene pairs become transitions with sequential `order_index = i+1` across the whole chapter. Final cut fetches `MovieShotTransition` filtered `video_url IS NOT NULL` ordered by `order_index`. Important: `_validate_transition_videos` only checks `video_url` presence â€” the `status != 'completed'` check is **commented out**, and because the query pre-filters null video_url, missing/failed transitions are silently dropped from the final cut rather than blocking it. Scene boundaries have no transition clip at all (hard cut between the last clip of scene N and first of scene N+1).

### 4. Subtitle + Ken Burns exact spec
`subtitle_service.py` builds time-windowed `drawtext` filters from Whisper word timings: `fontsize={size}:fontcolor={color}:borderw=5:bordercolor=black:shadowcolor=black@0.7:shadowx=4:shadowy=4:box=1:boxcolor=black@0.65:boxborderw=20:x=(w-text_w)/2:y={pos}:enable='between(t,start,end)'`. Text over `max_line_chars` splits into two lines at the nearest word boundary to the midpoint, line spacing `1.2 * font_size`, lines centered above/below base y. Ken Burns (`build_sentence_video_command`): `scale WxH force_original_aspect_ratio=decrease` + black pad, then `zoompan z='min(1+0.00015*on,1.15)'` with x/y drifting 5% of frame size across `total_frames = fps*duration`, `-loop 1` on the image, `-shortest` against the TTS audio.
